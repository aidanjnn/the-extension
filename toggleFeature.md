# Implementation Plan: Global Extension Toggle (Normal vs Modified Browser)

## Feature Overview
This is a "Master Killswitch" button in the Side Panel. When clicked, it instantly turns off all generated Browser Forge extensions so the user can quickly view the "normal" browser. Clicking it again turns them all back on. It includes a 10-second cooldown to prevent spamming.

This is different from the **Live Preview** feature:
- *Live Preview* is for testing a **single** new extension's code *before* it is formally installed.
- *Global Toggle* is for turning **all installed** modifications on/off globally across the entire browser.

---

## 1. Add the `management` Permission
**File:** `browser-agent-console/manifest.config.ts`

**Context:** To control other extensions, the Browser Forge side panel needs the Chrome Management API permission.
**Action:** Add `'management'` to the `permissions` array.

```typescript
  permissions: [
    'sidePanel',
    'storage',
    'tabs',
    'scripting',
    'management' // <-- Add this
  ],
```

## 2. Build the Toggle Logic
**File:** `browser-agent-console/src/sidepanel/App.tsx`

**Context:** We will query `chrome.management.getAll()` to find all extensions generated by our app, and flip their `enabled` state.

**Action:** Add the toggle state, cooldown timer, and the toggle function inside your React component.

```typescript
const [isNormalMode, setIsNormalMode] = useState(false);
const [toggleCooldown, setToggleCooldown] = useState(0);

const toggleAllExtensions = async () => {
  if (toggleCooldown > 0) return; // Prevent spamming

  // 1. Get all installed Chrome extensions
  const extensions = await chrome.management.getAll();

  // 2. Filter for only the extensions generated by Browser Forge
  // (You can identify them by checking if description contains "Generated extension")
  const generatedExtensions = extensions.filter(ext => 
    ext.description?.includes("Generated extension for:")
  );

  // 3. Determine the new state (if currently normal mode, turn them back ON)
  const turningOn = isNormalMode;

  // 4. Flip the state for all generated extensions
  for (const ext of generatedExtensions) {
    if (ext.enabled !== turningOn) {
      await chrome.management.setEnabled(ext.id, turningOn);
    }
  }

  // 5. Update UI state
  setIsNormalMode(!isNormalMode);

  // 6. Start the 10-second spam cooldown
  setToggleCooldown(10);
  const interval = setInterval(() => {
    setToggleCooldown((prev) => {
      if (prev <= 1) {
        clearInterval(interval);
        return 0;
      }
      return prev - 1;
    });
  }, 1000);
  
  // 7. Refresh the active tab so they instantly see the changes
  if (currentActiveTab?.id) {
    chrome.tabs.reload(currentActiveTab.id);
  }
};
```

---

## 3. Add the UI Button
**File:** `browser-agent-console/src/sidepanel/App.tsx` (and `App.css`)

**Action:** Add a "Power" or "Toggle" button to the top header of the side panel.

```tsx
<button 
  className={`global-toggle-btn ${isNormalMode ? 'normal-mode' : 'modified-mode'}`}
  onClick={toggleAllExtensions}
  disabled={toggleCooldown > 0}
>
  {toggleCooldown > 0 
    ? `Wait ${toggleCooldown}s...` 
    : (isNormalMode ? "Turn Extensions ON" : "View Normal Browser")
  }
</button>
```

**Why this feature is incredibly fast:**
Because we are using `chrome.management.setEnabled()`, Chrome handles the toggle natively in milliseconds without invoking the slow OS AppleScript macros. Mated with an automatic tab refresh, the user will instantly jump back and forth between the "Vanilla" internet and their fully customized internet!
